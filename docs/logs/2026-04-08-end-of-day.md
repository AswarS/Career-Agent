## What Was Learned Today

### 1. Shell work and workflow work should stay split

It was correct to keep today focused on shell behavior.
Trying to mix conversation-driven workflow into the same iteration would have
blurred whether a problem came from layout or from interaction state.

### 2. Collapsed rail design needed real design, not abbreviation compression

The first collapsed rail technically worked but felt wrong.
The accepted solution was to treat the compact rail as its own navigation mode.

### 3. Simulations should prefer immersive in-app canvas over new windows

This is now both a product decision and an implemented shell capability.
Detached windows remain a possible later enhancement, not the MVP default.

### 4. Direct-to-main is acceptable for very small, low-risk polish or docs

That exception is now documented.
It should not be used for workflow logic, adapter changes, or broader state work.
