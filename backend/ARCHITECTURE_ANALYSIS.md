# Claude Code 架构深度分析文档

> 基于第一性原理的源码架构解析

---

## 一、项目概述

**Claude Code** 是 Anthropic 官方推出的命令行 AI 编程助手工具。本项目是其从 source maps 重建的源码树，版本 2.1.88。

**核心定位**：一个运行在终端的智能编程代理，能够理解代码、执行命令、编辑文件、并与 AI 模型进行交互式对话。

---

## 二、第一性原理：系统存在的根本目的

### 2.1 核心问题
程序员在开发过程中需要：理解代码库、执行重复操作、调试问题、编写代码。传统方式依赖人工逐一完成，效率低下。

### 2.2 解决方案
构建一个能够：
1. **感知** — 读取文件、理解代码结构、执行命令
2. **思考** — 通过 AI 模型分析问题、制定计划
3. **行动** — 编辑文件、运行命令、创建提交
4. **交互** — 与用户对话、请求确认、反馈结果

### 2.3 设计约束
- 必须在终端环境运行（CLI 优先）
- 必须安全可控（权限系统）
- 必须支持扩展（MCP 协议、插件系统）
- 必须高效响应（启动优化、缓存机制）

## 四、核心模块详解

### 4.1 启动流程（Bootstrap）

**入口文件**：`src/bootstrap-entry.ts`

启动遵循「渐进式加载」原则：

1. **零依赖快路径** — `--version` 等简单命令直接输出，无需加载任何模块
2. **预启动优化** — 并行执行 MDM 配置读取、钥匙串预取等耗时操作
3. **完整初始化** — 加载 CLI 主入口 `src/entrypoints/cli.tsx`

**设计意图**：终端工具必须快速响应。通过延迟加载和并行预取，将启动时间从几百毫秒压缩到几十毫秒。

---

### 4.2 命令系统（Commands）

**目录**：`src/commands/`

Claude Code 内置丰富的斜杠命令体系：

| 命令类别 | 示例 | 功能 |
|---------|------|------|
| 会话管理 | `/resume`, `/clear` | 恢复/清除会话 |
| 代码操作 | `/commit`, `/pr`, `/review` | Git 工作流 |
| 配置管理 | `/config`, `/model`, `/permissions` | 设置调整 |
| 工具集成 | `/mcp`, `/ide`, `/hooks` | 外部集成 |
| 调试辅助 | `/doctor`, `/cost`, `/compact` | 诊断工具 |

**实现模式**：每个命令是独立模块，导出统一的 `Command` 对象，包含：
- 名称和描述
- 参数定义
- 执行函数
- 权限要求

---

### 4.3 工具系统（Tools）

**核心定义**：`src/Tool.ts`

工具是 AI 与外界交互的「手脚」。每个工具实现：

```
工具接口 = {
  名称: 唯一标识
  输入模式: 参数验证规则
  执行函数: 具体操作逻辑
  权限检查: 是否允许执行
  进度回调: 执行状态反馈
}
```

**内置工具清单**（部分）：

| 工具名 | 功能 | 风险等级 |
|-------|------|---------|
| Bash | 执行 Shell 命令 | 高 |
| FileRead | 读取文件内容 | 中 |
| FileEdit | 编辑文件 | 高 |
| FileWrite | 创建/覆盖文件 | 高 |
| Glob | 文件模式匹配 | 低 |
| Grep | 内容搜索 | 低 |
| WebFetch | 获取网页内容 | 中 |
| WebSearch | 网络搜索 | 中 |
| Agent | 启动子代理 | 高 |
| Skill | 执行技能脚本 | 中 |
| Task* | 任务管理系列 | 中 |

**设计亮点**：
- 工具可动态加载（条件编译）
- 支持工具预设（tool preset）
- 工具权限独立控制

---

### 4.4 权限系统（Permissions）

**目录**：`src/utils/permissions/`

这是 Claude Code 最关键的安全子系统。

#### 权限模式

| 模式 | 说明 |
|-----|------|
| `default` | 默认模式，敏感操作需确认 |
| `acceptEdits` | 自动接受文件编辑 |
| `bypassPermissions` | 跳过所有权限检查（危险） |
| `plan` | 计划模式，只读操作 |
| `auto` | 自动模式（需要 classifier 判断） |

#### 权限规则结构

```
权限规则 = {
  来源: userSettings | projectSettings | localSettings | policySettings | ...
  行为: allow | deny | ask
  目标: { 工具名, 规则内容 }
}
```

示例：允许所有读取操作，但禁止删除命令：
- allow: Read(*)
- deny: Bash(rm *)

#### 权限决策流程

```
用户请求 → 检查权限规则 → 匹配?
                          ├─ allow → 直接执行
                          ├─ deny → 直接拒绝
                          └─ ask/no match → 弹出确认对话框
```

#### 自动模式分类器

`yoloClassifier.ts` 实现了一个基于规则和 AI 的分类器，用于判断命令是否安全可自动执行：
- 危险命令模式匹配（如 `rm -rf`）
- 敏感路径检测
- AI 辅助判断（可选）

---

### 4.5 查询引擎（QueryEngine）

**核心文件**：`src/QueryEngine.ts`

这是整个系统的「大脑」，负责：

1. **消息管理** — 维护对话历史，处理压缩
2. **API 调用** — 与 Claude API 通信，处理流式响应
3. **工具调度** — 解析工具调用请求，执行并返回结果
4. **错误处理** — 重试机制、降级策略

#### 核心流程

```
用户输入
   ↓
构建消息 (系统提示 + 对话历史 + 用户消息)
   ↓
调用 Claude API
   ↓
接收流式响应
   ├─ 文本内容 → 直接显示
   └─ 工具调用 → 执行工具 → 追加结果 → 继续请求
   ↓
生成完整响应
```

#### 上下文压缩

当对话历史过长时，触发压缩机制：
- 保留关键信息
- 移除冗余内容
- 维护对话连贯性

---

### 4.6 状态管理（State）

**核心文件**：`src/state/AppState.tsx`、`src/state/AppStateStore.js`

采用 React 状态管理方案：

```
AppState = {
  会话状态: 消息列表、当前模型、权限上下文
  UI 状态: 输入缓冲、光标位置、滚动位置
  工具状态: 正在执行的工具、进度信息
  设置状态: 用户偏好、项目配置
}
```

**设计特点**：
- 使用 Zustand-like store 模式
- 支持状态快照和恢复
- 响应式更新 UI

---

### 4.7 MCP 协议集成

**目录**：`src/services/mcp/`

MCP (Model Context Protocol) 是 Anthropic 推出的模型上下文协议，允许外部工具与 AI 模型标准化交互。

**支持的传输方式**：
- Stdio — 标准输入输出（本地进程）
- SSE — Server-Sent Events（HTTP 长连接）
- Streamable HTTP — 可流式 HTTP

**MCP 工具注册流程**：
1. 读取 MCP 服务器配置
2. 建立连接（启动进程或发起 HTTP）
3. 发现工具列表
4. 注册为 Claude Code 内置工具
5. 代理工具调用请求

---

### 4.8 设置系统（Settings）

**目录**：`src/utils/settings/`

多层配置来源，优先级从高到低：

```
CLI 参数 > 环境变量 > 会话设置 > 项目设置 > 用户设置 > 策略设置 > MDM 设置
```

**配置文件位置**：
- 用户级：`~/.claude/settings.json`
- 项目级：`.claude/settings.json`
- 策略级：`/etc/claude/managed-settings.json`（企业部署）

**支持热更新**：配置变更无需重启，实时生效。

---

### 4.9 Agent 系统

**目录**：`src/tools/AgentTool/`

Agent 是一个独立的子代理实例，可以：
- 执行独立任务
- 拥有独立的工具集
- 支持并行执行多个 Agent

**Agent 类型**：
- 通用代理 — 处理任意任务
- 专用代理 — 针对特定领域优化
- 后台代理 — 无 UI 的后台任务

**协同模式**：
主代理可以启动多个子代理并行工作，然后汇总结果。

---

### 4.10 技能系统（Skills）

**目录**：`src/skills/`

技能是预定义的任务模板，可以是：
- Shell 脚本
- TypeScript 模块
- MCP 工具封装

**内置技能**：
- `/commit` — 智能提交
- `/review-pr` — PR 审查
- `/update-config` — 配置更新

**扩展机制**：用户可在 `.claude/skills/` 目录创建自定义技能。

## 五、关键技术决策

### 5.1 为什么选择 Bun 而非 Node.js？

- **更快的启动速度** — Bun 的模块加载速度是 Node 的 2-3 倍
- **内置功能** — 支持 JSX、TypeScript 无需配置
- **原生 API** — `Bun.file()` 等高性能 I/O
- **条件编译** — `feature('FLAG')` 实现编译时特性开关

### 5.2 为什么使用 React/Ink 构建 CLI？

- **组件化 UI** — 复杂交互逻辑模块化
- **声明式渲染** — 状态驱动的界面更新
- **丰富的 hooks** — 复用逻辑（如 `useTextInput`）

### 5.3 权限系统的设计哲学

**最小权限原则**：默认拒绝，按需授权。

**分层控制**：
1. 全局规则 — 企业策略
2. 项目规则 — 团队约定
3. 会话规则 — 临时授权

**可审计性**：所有权限决策都有日志记录。

## 七、扩展机制

### 7.1 添加新工具

1. 在 `src/tools/` 创建工具目录
2. 实现工具接口
3. 在 `src/tools.ts` 注册
4. 添加权限规则

### 7.2 添加新命令

1. 在 `src/commands/` 创建命令模块
2. 导出 `Command` 对象
3. 在 `src/commands.ts` 导入

### 7.3 集成 MCP 服务器

1. 在设置中添加 MCP 服务器配置
2. 指定启动命令或 URL
3. Claude Code 自动发现工具

## 八、安全考量

### 8.1 敏感数据处理

- API Key 存储在系统钥匙串（macOS Keychain / Windows Credential Manager）
- 环境变量中的敏感信息不会出现在日志
- 对话历史本地存储，可选加密

### 8.2 命令注入防护

- Bash 命令需要用户确认（除非明确授权）
- 危险模式检测（如 `rm -rf /`）
- 路径遍历保护

### 8.3 网络安全

- 仅 HTTPS 通信
- 证书验证
- 代理支持

---

## 九、性能优化策略

### 9.1 启动优化

- 并行预取（钥匙串、MDM 配置）
- 延迟加载非必要模块
- 快路径提前返回

### 9.2 运行时优化

- 文件状态缓存
- 增量式上下文压缩
- 工具执行结果缓存

### 9.3 内存管理

- 会话历史压缩
- 大文件流式处理
- 及时释放资源

## 十、总结

Claude Code 是一个设计精良的终端 AI 编程助手，其架构体现了以下设计原则：

1. **模块化** — 各子系统职责清晰，易于维护和扩展
2. **安全优先** — 完善的权限系统，多层防护
3. **性能导向** — 启动优化、缓存机制、异步执行
4. **可扩展** — MCP 协议、插件系统、技能机制
5. **用户体验** — 丰富的命令、直观的交互、详细的反馈

通过本分析，应该能够对整个系统的运作机制有清晰的理解，为进一步的定制开发或问题排查提供基础。
